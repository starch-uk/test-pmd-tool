# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in this project, please report it to **security@starch.uk**. We take security seriously and will respond promptly.

## Supported Versions

We actively support the following versions with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Security Best Practices

### For Contributors

- Run `pnpm audit` before committing changes
- Review dependencies for known vulnerabilities
- Use secure coding practices
- Validate input and avoid injection vulnerabilities

### For Users

- Keep dependencies updated
- Use the latest stable version
- Report suspicious behavior
- Validate rule files before testing

## Disclosure Process

1. **Report**: Send details to security@starch.uk
2. **Acknowledgment**: We'll acknowledge receipt within 48 hours
3. **Investigation**: We'll investigate and provide updates
4. **Fix**: We'll develop and test a fix
5. **Disclosure**: We'll coordinate disclosure of the vulnerability

## Recognition

Security researchers who responsibly disclose vulnerabilities will be recognized in our Hall of Fame (when established) unless they prefer to remain anonymous.
